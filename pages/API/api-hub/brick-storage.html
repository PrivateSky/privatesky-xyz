The Brick Storage component of VirtualMQ allows the storage and retrieval of bricks (blocks of data from a file). As a brick is uniquely identified by the cryptographic hash of its content, any brick stored in VirtualMQ is an unique block of data. Also, when receiving a brick, VirtualMQ computes the hash of the data and compares it with the brick id included in the request parameters and if the two do not coincide, the data is discarded and the an error is thrown.
