PSK Web Server is a HTTP/HTTPS server that allows the communication between PrivateSky nodes and/or agents inside a node, allows the storage and retrieval of bricks (blocks of data stored on the server in files identified by the block's cryptographic hash) and allows the grouping of different bricks under the same unique identifier (anchor). Each one of the three components is explained in more detail at the links below. The PSK Web Server can also be extended to include custom services (besides the default ones below). The section How to extend a PSK Web Server explains in more detail how a PSK Web Server can be extended. Brick Storage Anchoring VirtualMQ Dossier Wizard A PSK Web Server is configured using the configuration file server.json located at ./privatesky/conf. If the configuration file does not exist, then a default configuration is used. Also, if the configuration file is missing fields necessary for the server to start, then the missing configurations are taken from the default configuration. Below is described the default PSK Web Server configuration. { "storage": path.join(process.env.PSK_ROOT_INSTALATION_FOLDER, "tmp"), "sslFolder": path.join(process.env.PSK_ROOT_INSTALATION_FOLDER, "conf", "ssl"), "port": 8080, "zeromqForwardAddress": "tcp://127.0.0.1:5001", "preventRateLimit": false, "endpoints": ["virtualMQ", "filesManager", "edfs", "dossier-wizard"], "endpointsConfig": { "virtualMQ": { "path": "./modules/psk-webserver/ChannelsManager.js", "channelsFolderName": "channels", "maxSize": 100, "tokenSize": 48, "tokenHeaderName": "x-tokenHeader", "signatureHeaderName": "x-signature", "enableSignatureCheck": true }, "dossier-wizard": { "path": "dossier-wizard" }, "edfs": { "path": "edfs-middleware" }, "filesManager": { "path": "./modules/psk-webserver/FilesManager.js" } } }; storage The folder in which PSK Web Server stores its files. By default, the server uses the folder ./privatesky/tmp (the environment variable PSK_ROOT_INSTALLATION_FOLDER stores path of the folder in which PrivateSky is installed). For the server to use another folder for storage, it is necessary to add the folder's path in the the configuration file's (server.json) storage property. If we want to set the server's storage folder to /home/bar, then the configuration file should be { "storage": "/home/bar" } Note: In order to change one or just a few properties in the default configuration it is sufficient to include just these properties in the configuration file, as the server takes the remaining properties from the default configuration presented above. For example, if the above configuration file is used, then the server will take all of the properties in the default configuration, except the storage property which is read from the configuration file. sslFolder The folder in which Psk Web Server looks for SSL certificates when started. If the folder contains certificates, then the PSK Web Server is started as a HTTPS server; otherwise, PKS Web Server is a HTTP server. By default, PSK Web server is a HTTP server (sslFolder is empty). If SSL certificates are copied into the sslFolder, then Psk Web Server is started as a HTTPS server.

port The port that the server uses to listen for requests. By default, the port 8080 is used. Below is an example of a configuration file that sets the server's port to 9090. { "port": 9090 } endpoints An array which contains the names of the services that the server provides. By default, the server provides four services: filesManager for uploading and downloading files, ChannelsManager which allows the communication between PrivateSky nodes and between processes running in the same node, edfs which allows the storage and retrieval of bricks (encrypted blocks of data identifiable by their content's cryptographic hash) and dossier-wizard which allows the creation and manipulation of dossiers. New services can be added by adding their name in endpoints and adding a corresponding entry in endpointsConfig. Note: Every service declared in endpoints should have a corresponding entry in endpointsConfig. Below is an example of a configuration file for a server that uses only one service, myService. As pointed out above, endpointsConfig contains the entry myService whose path property is set to "/home/bar", the file system path where the module that contains myService's implementation is located. { "endpoints": ['myService'], "endpointsConfig": { "myService": { "path": "/home/bar/myService" } } } endpointsConfig An object that contains the service's configurations. Each service declared in endpoints should have the path property endpointsConfig set. The path property is either a file system path to a module that contains the service or the name of a PrivateSky module. In the default configuration the edfs service has the path edfs-middleware which is the name of a PrivateSky module and channelsManager's path property is ./modules/psk-webserver/ChannelsManager which is a file system path. The createPskWebServer() method creates and starts a VirtualMQ server. createPskWebServer(port, rootFolder, sslConfig, callback) port The port on on which the server is listening for requests.

rootFolder The folder where the server keeps the files uploaded on it.

sslConfig |Optional An object that configures, among others, the communication protocol used to communicate with clients. By default, "http" is used.

callback The function called when the server is started.

 The server instance. The createPskWebServer() method starts a PSK Web Server and returns its instance. In the following example, a server that listens on port 8080 and saves files in folder "/foo/tmp" is created. const VirtualMQ = require("virtualmq"); const port = 8080; const server VirtualMQ.createPskWebServer(port, "/foo/tmp", undefined, (err) => { if (err) { throw err; } console.log("The server was started and is listening on port", port); }); The createPskWebServer() method starts a HTTP/HTTPS server and return the server instance. As a result, the server can be dynamically extended using the API below. server.use(url, handler); url The url where the server listens for requests. handler The function called for filtering requests that arrive at the server on the specified url. server.put(url, handler); url The url where the server listens for PUT requests. handler The function called when the server receives a PUT request at the specified url. server.post(url, handler); url The url where the server listens for POST requests. handler The function called when the server receives a POST request at the specified url. server.get(url, handler); url The url where the server listens for GET requests. handler The function called when the server receives a GET request at the specified url.
