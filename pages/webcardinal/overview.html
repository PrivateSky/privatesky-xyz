WebCardinal represents the complete rewrite version of Cardinal, with the aim to enable an even more rapid development of WEB applications. WebCardinal offers almost all the functionalities of Cardinal, either is the same form or by providing an easier alternative.

Since WebCardinal is a complete rewrite of Cardinal, there may be some cases in which backward compatibility is not offered. One such good example of limited compatibility is with Cardinal's psk-modal, which is not at all supported in the new version. Instead, the improved webc-modal was created.
Read more about it in its dedicated documentation. The current webpage is a working example of a WEB application written using WebCardinal and including also Cardinal components.

Any known incompatibilities with Cardinal are explained when applicable.

 WebCardinal offers the same configuration possibilities as Cardinal, while offering some extra ones.
Read more about it in the documentation for Configuration. WebCardinal offers some useful components to make the development of WEB applications even more easier.
Read more about each webcardinal-core component by checking the webcardinal-core package section from the WebCardinal Cheatsheet. WebCardinal comes with some very useful and powerful attributes that can be used on mainly every HTML element. data-for - improved and optimized version of the psk-for-each component data-if - improved and optimized version of the psk-condition component data-model - used for binding many tag attributes to the same element data-tag - used for an easier event subcription experience Considering the fact that Cardinal's psk-* components handle their own logic, flow and HTML manipulation, not all of them can integrate perfectly with the new attributes of WebCardinal. So extra care needs to be taken in this case.

WebCardinal's approach to controllers is similiar with the one from Cardinal.

Read more about Cardinal's controllers here

Currently, WebCardinal offers the following Controllers which can be extended based on the given requirements:

Controller: offers the basic functionalities of the framework WebController: offers some basic interaction with the WebCardinal components

The controlers can be used from WebCardinal.controllers available on the window object.

const { WebcController } = WebCardinal.controllers; class Page1Controller extends WebcController { getModel = (_) => ({ field1: 1, field2: 2 }); constructor(element, history) { super(element, history); this.setModel(this.getModel()); } }

WebCardinal's approach to models is similiar with the one from Cardinal.

Read more about Cardinal's models here

WebCardinal went further and added a more standardised and consistent approach to models and model binding.

All of the HTML element and also custom elements have support for binding. One-way binding is ensured for every element, but there may be some limitations for two-way binding when using non-native HTML form elements. Read more about WebcCardinal's forms here.

In order to offer a consistent approach for binding, WebCardinal has support for a special attribute data-model.

Read more about WebCardinal's data-model attribute here

WebCardinal has added a new syntax that allows model binding (and also translation binding) inside the template nodes.

Message: {{ @hello }}

 const { WebcController } = WebCardinal.controllers; export default class BindingExampleController extends WebcController { getModel = (_) => ({ buttonText: "Button", hello: "Hello world!", }); constructor(element, history) { super(element, history); this.setModel(this.getModel()); } }
